📋 POST-INSTALLATION SETUP
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. LOGIN TO WORDPRESS ADMIN
   https://yourdomain.com/wp-admin/

2. NAVIGATE TO PLUGINS
   Plugins → Installed Plugins → LounGenie Portal → Activate

3. CREATE FIRST SUPPORT USER
   Users → Add New
   • Username: support_user
   • Email: support@company.com
   • Role: LounGenie Support Team
   • Password: [strong password]
   • Click: Create User

4. CREATE FIRST PARTNER USER (OPTIONAL)
   Users → Add New
   • Username: partner_user
   • Email: partner@company.com
   • Role: LounGenie Partner Company
   • Password: [strong password]
   • Click: Create User
   • Edit user → Meta field "lgp_company_id" = 1 (or company ID)

5. ACCESS THE PORTAL
   https://yourdomain.com/portal
   Login with support or partner user

6. CONFIGURE INTEGRATIONS (OPTIONAL)
   WordPress Admin → Settings → [Integration Settings]
   • Email integration
   • HubSpot CRM
   • Microsoft 365 SSO

🆘 TROUBLESHOOTING
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Problem: Plugin won't activate
Solution: 
  • Check PHP version (7.4+ required)
  • Check file permissions (755 for folders)
  • Check wp-content/debug.log
  • Try reactivating
  • Contact hosting provider

Problem: Portal page shows blank
Solution:
  • Go to Settings → Permalinks → Save
  • Clear browser cache (Ctrl+Shift+Delete)
  • Check error logs
  • Verify /portal route exists
  • Check plugin is activated

Problem: Database errors
Solution:
  • Verify database user has CREATE TABLE permission
  • Check database character set (UTF-8)
  • Verify sufficient disk space
  • Run migrations: wp plugin activate loungenie-portal
  • Check error logs

Problem: Users can't login
Solution:
  • Verify user has correct role assigned
  • Check user meta lgp_company_id
  • Clear browser cache
  • Check error logs
  • Verify SSL certificate (if HTTPS)

See DEPLOYMENT_CHECKLIST.md in plugin for more
