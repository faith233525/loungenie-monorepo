HOW THE PLUGIN WORKS (QUICK SUMMARY)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. ACTIVATION
   ├─ loungenie-portal.php is loaded by WordPress
   ├─ Plugin header is read
   └─ Activation hook triggers database setup

2. BOOTSTRAP (LGP_Loader)
   ├─ PHASE 1: Database, Security, Capabilities
   ├─ PHASE 2: Auth, Router, Assets
   ├─ PHASE 3: Logging, Notifications, REST APIs
   └─ PHASE 4: Features, Email, SSO, etc.

3. REQUEST ROUTING
   ├─ User visits /portal/* URL
   ├─ LGP_Router matches the route
   ├─ Loads appropriate template
   └─ Portal-shell.php wraps everything

4. AUTHENTICATION
   ├─ Check if user logged in (is_user_logged_in)
   ├─ Check role (is_support or is_partner)
   ├─ Filter data by role/company_id
   └─ Log auth events (audit trail)

5. API CALLS
   ├─ Frontend JavaScript calls /lgp/v1/* endpoints
   ├─ Nonce verified for CSRF protection
   ├─ User permission checked
   ├─ Data queried with prepared statements
   └─ Response returned via rest_ensure_response

6. RENDERING
   ├─ Portal-shell.php loads main wrapper
   ├─ Sub-template rendered (tickets, dashboard, etc.)
   ├─ All HTML scoped under .lgp-portal
   ├─ All output escaped (esc_html, esc_url, esc_attr)
   └─ CSS/JS properly enqueued
