The plugin checks for configuration in this priority order:

  1. PHP Constants (wp-config.php)      ← Highest priority
       ↓ (if not defined)
  2. Environment Variables (.env)
       ↓ (if not set)
  3. WordPress Options (admin UI)
       ↓ (if not configured)
  4. Default Values (safe fallbacks)    ← Lowest priority

This means:
  • Admin UI configuration works out-of-the-box
  • wp-config.php constants override admin settings
  • Environment variables work for containers
  • Users can choose their preferred method
